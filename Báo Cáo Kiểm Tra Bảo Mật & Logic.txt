### 🔴 BM-02: Telegram Bot không kiểm tra nguồn gốc tin nhắn trước khi xử lý

**File:** `telegramService.js`, dòng 24–30

Hệ thống chỉ kiểm tra `UserModel.findByChatId` nhưng **không giới hạn tốc độ (rate limit)** tại tầng Telegram. Nếu một user hợp lệ bị lộ `chatId` hoặc bot bị forward, kẻ tấn công có thể spam giao dịch hàng loạt.

**Thêm nên:**
```js
// Theo dõi số tin nhắn / phút per chatId
const rateLimitMap = new Map();
bot.on('message', async (msg) => {
    const now = Date.now();
    const key = msg.chat.id;
    const last = rateLimitMap.get(key) || [];
    const recent = last.filter(t => now - t < 60000); // trong 60s
    if (recent.length >= 10) {
        bot.sendMessage(key, '⚠️ Bạn gửi quá nhiều tin nhắn. Vui lòng đợi.');
        return;
    }
    rateLimitMap.set(key, [...recent, now]);
    // ... xử lý tiếp
});
```

### 🔴 BM-03: Tiêm lệnh tiềm năng qua `ai_storage_path`

**File:** `aiManager.js`, dòng 13–21

```js
const customPath = SettingModel.get('ai_storage_path');
const rootDir = customPath || path.join(__dirname, '../');
this.enginePath = path.join(this.binDir, 'llama-server.exe');
// ...
this.process = spawn(this.enginePath, [], { env });
```

`ai_storage_path` được lưu từ API (`POST /api/ai/settings`) mà không có kiểm tra whitelist hoặc validate định dạng đường dẫn. Nếu kẻ tấn công có quyền Admin và muốn ghi path dạng `../../malicious/`, `spawn()` sẽ chạy file bất kỳ trên server.

**Cần thêm:**
```js
function validateStoragePath(inputPath) {
    const resolved = path.resolve(inputPath);
    // Chỉ cho phép path tuyệt đối trỏ đến thư mục an toàn
    if (!path.isAbsolute(resolved)) throw new Error('Path phải là tuyệt đối');
    if (resolved.includes('..')) throw new Error('Path không hợp lệ');
    return resolved;
}
```

### 🟠 BM-04: CORS cho phép `null` origin (Electron bypass)

**File:** `server.js`, dòng 28–37

```js
if (!origin || origin === 'null' || allowed.includes(origin)) {
    callback(null, true);
}
```

`origin === 'null'` được cho phép để hỗ trợ Electron, nhưng **mọi request từ `file://`** (kể cả file HTML độc hại chạy locally) đều vượt qua CORS. Nên dùng header tùy chỉnh hoặc kiểm tra `User-Agent` để phân biệt Electron thật.

### 🟠 BM-05: `aiManager.downloadFile` không kiểm tra URL

**File:** `aiManager.js`, dòng 45–58

URL của engine và model được hardcode, tuy nhiên nếu sau này cho phép user cấu hình URL tải model tùy chỉnh (tính năng có thể mở rộng), không có validation nào. Cần whitelist domain:

```js
const ALLOWED_DOMAINS = ['huggingface.co', 'github.com', 'ggml-org.github.io'];
function validateDownloadUrl(url) {
    const parsed = new URL(url);
    if (!ALLOWED_DOMAINS.includes(parsed.hostname)) throw new Error('Domain không được phép');
}
```

### 🟡 BM-07: Rate limit áp dụng cho cả `/api/auth` — dễ bị lockout hợp lệ

**File:** `server.js`, dòng 43–47

```js
app.use('/api/', limiter); // áp dụng cho tất cả /api/
```

Rate limit 100 req/15 phút áp dụng cả cho login và các endpoint khác chung một IP. Trong môi trường văn phòng (nhiều user cùng IP NAT), 100 request là quá ít và có thể chặn nhầm user hợp lệ. Nên tách rate limit riêng cho `/api/auth` với giới hạn khắt khe hơn (ví dụ 10 lần/15 phút).

## PHẦN 2 — LOGIC AI

### 🟠 LG-01: `anomalyDetector.js` không được dùng trong pipeline chính

**File:** `ai.js` (cả `/analyze-all` và `/re-analyze`)

Module `anomalyDetector.js` đã cài đặt `stdDev`, `mean`, và ngưỡng chuẩn thống kê — nhưng **không được gọi ở đâu trong các endpoint phân tích**. Cả 2 endpoint tự tính `avg` thủ công mà thiếu `stdDev`, dẫn đến phát hiện bất thường kém chính xác hơn so với công cụ đã có sẵn.

```js
// HIỆN TẠI (trong /re-analyze và /analyze-all)
const avg = historical.reduce((s, r) => s + r.amount, 0) / historical.length;
if (t.amount > avg * 3.0) { ... }

// NÊN SỬA
const anomalyDetector = require('./anomalyDetector');
const result = anomalyDetector.detectAnomalies(t.category_id, t.amount, t.type);
if (result.isAnomaly) {
    explanation = `Cảnh báo: Lệch ${result.deviation} so với trung bình...`;
}
```

## PHẦN 3 — LOGIC TELEGRAM & PHÂN LOẠI

### 🟠 LG-07: Telegram fallback regex có thể parse sai số tiền lớn

**File:** `telegramService.js`, dòng 75–84

```js
const amountRegex = /(?:thu|chi)?\s*([\d,\.]+)\s*(k|tr|m|t)?\s*(.*)/i;
```

Regex này match cả `t` (terabyte) lẫn `tr` (triệu). Ví dụ `"Chi 5t xăng"` có thể bị hiểu là 5 triệu hoặc 5 nghìn tùy thứ tự match. Nên dùng regex cụ thể hơn và xử lý theo độ ưu tiên:

```js
// Ưu tiên: tr > t > k > m (tránh nhầm)
const suffixMap = { 'tr': 1_000_000, 'triệu': 1_000_000, 'm': 1_000_000, 'k': 1_000, 't': 1_000 };
```

Mọi giao dịch từ Telegram đều được phân vào scope `business`, kể cả chi tiêu cá nhân. Nếu hệ thống có scope `personal`, user không có cách chọn. Cần cho phép user cấu hình default scope trong profile hoặc nhận diện từ text (ví dụ: "chi cá nhân 50k" → scope personal).

## PHẦN 4 — QUẢN LÝ TIẾN TRÌNH & ĐỘ ỔN ĐỊNH

### 🟠 LG-11: `aiManager.startServer()` không có timeout hoặc health-check

**File:** `aiManager.js`, dòng 80–100

Sau khi `spawn()`, không có cơ chế kiểm tra xem engine đã thực sự sẵn sàng chưa. Endpoint `/chat` có thể nhận request khi engine vẫn đang khởi động và trả về lỗi 503 không rõ ràng cho user.

**Nên thêm:** Polling `/health` của llama-server sau khi spawn:
```js
async waitForEngine(timeoutMs = 30000) {
    const start = Date.now();
    while (Date.now() - start < timeoutMs) {
        try {
            await fetch('http://127.0.0.1:1234/health');
            return true; // Engine sẵn sàng
        } catch { await new Promise(r => setTimeout(r, 1000)); }
    }
    return false;
}
```

### 🟠 LG-12: `aiManager.startServer()` auto-start không có guard trên non-Windows

**File:** `server.js`, dòng 60–63; `aiManager.js`, dòng 82

```js
this.enginePath = path.join(this.binDir, 'llama-server.exe'); // hardcode .exe
// ...
if (specs.engineExists && specs.modelExists) {
    aiManager.startServer(); // spawn .exe trên Linux/Mac → crash im lặng
}
```

Nếu deploy lên Linux server (VPS, Docker), `spawn('llama-server.exe')` sẽ thất bại im lặng. Nên kiểm tra platform:

```js
const exeName = process.platform === 'win32' ? 'llama-server.exe' : 'llama-server';
this.enginePath = path.join(this.binDir, exeName);
```

### 🟡 LG-13: `analyze-all` chạy transaction DB đồng bộ có thể block event loop

**File:** `ai.js`, dòng 287–326

Hàm `db.transaction()` (better-sqlite3) chạy đồng bộ, xử lý **toàn bộ confirmed transactions** trong một lần. Nếu có hàng nghìn giao dịch, Node.js event loop sẽ bị block trong thời gian dài, ảnh hưởng mọi request khác.

**Giải pháp:** Chia batch với `setImmediate` hoặc chỉ phân tích các giao dịch mới (`WHERE ai_explanation IS NULL`).
